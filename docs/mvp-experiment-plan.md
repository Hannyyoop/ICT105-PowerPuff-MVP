# MVP Experiment Plan

## 1. Group and Project Information

- **Group Name:** Powepuff
- **Project Title:** University Room Booking System
- **Repository Link:** https://github.com/Hannyyoop/ICT105-PowerPuff-MVP
- **Main Target User:** University Students and Staff
- **Prototype Platform:** Frontend-only (HTML, CSS, JavaScript with Local Storage)

---

## 2. Experiment Objective

> We want to test whether students can successfully view available rooms, search for a room, book a room, and manage their bookings, while allowing staff to manage room information through a simple web prototype without requiring a backend.

---

## 3. Requirement Scope for the Experiment

| Requirement ID | Requirement Summary | Related Screen/Feature | Tested in This Experiment? |
|---|---|---|---|
| FR-01 | Display portal/homepage | Homepage | Yes |
| FR-02 | View available rooms | Rooms Page | Yes |
| FR-03 | Search available rooms | Rooms Page | Yes |
| FR-04 | Book a room | Booking Page | Yes |
| FR-05 | View booking history | My Bookings | Yes |
| FR-06 | Cancel a booking | My Bookings | Yes |
| FR-07 | Manage room information | Admin Page | Yes |
| FR-08 | View booking summary | Dashboard | Yes |

---

## 4. MVP Experiment Type

### Selected Experiment Type

- Clickable Prototype
- Simple Web Prototype
- Dashboard Demo

### Reason for Selection

The project is implemented as a frontend-only web application using HTML, CSS, JavaScript, and Local Storage. This allows users to interact with the main booking workflow while simulating room management and dashboard features without requiring a backend database.

---

## 5. Test Users

| Test User Group | Number of Testers | Why They Are Relevant |
|---|---:|---|
| Students | 5 | They are the primary users who will search and book available rooms. |
| Staff | 2 | They will evaluate the room management and dashboard features. |

---

## 6. Experiment Procedure Summary

1. Introduce the prototype and explain its purpose.
2. Ask participants to access the Student Portal.
3. Ask students to:
   - View available rooms.
   - Search for a room.
   - Book a room.
   - View their bookings.
   - Cancel a booking.
4. Ask staff members to:
   - Access the Admin Portal.
   - Add a new room.
   - Edit or delete a room.
   - Change a room's availability status.
   - View the dashboard summary.
5. Observe how easily users complete each task without assistance.
6. Collect feedback regarding usability, navigation, and overall user experience.

---

## 7. Expected Learning

After conducting the experiment, the team expects to determine whether users can complete the main room booking workflow easily and whether the current interface is intuitive. The feedback will help identify usability issues, improve navigation, refine room management features, and prioritize enhancements before developing a full backend version of the system.
